# BF6 Portal Tool

![Version](https://img.shields.io/badge/version-1.2.0-blue.svg)
![License](https://img.shields.io/badge/license-ISC-green.svg)

**BF6 Portal Tool** is a powerful, standalone visual logic editor for **Battlefield 6 Portal**, built using **Electron** and **Google Blockly**. It replicates the official Rules Editor experience in a desktop application, allowing for offline development, easier management of complex logic, and custom extensions.

## ?? Features

*   **Authentic Interface:** Fully recreated Toolbox menu structure matching the official Battlefield Portal editor (Rules, AI, Vectors, Gameplay, etc.).
*   **Visual Scripting:** Intuitive drag-and-drop block interface powered by Blockly.
*   **Comprehensive Block Library:** Includes hundreds of blocks for Game Modes, AI logic, Math, Event Payloads, and more.
*   **Validation:** Visual cues for Statement (Yellow) vs. Value (Green) block connections.
*   **Desktop Power:** Runs as a native application on Windows (Electron-based).
*   **Maintenance Tools:** Includes a suite of Python scripts to auto-generate and update block definitions from data sources.

### Prerequisites

*   **Node.js** (v18 or higher)

### Getting Started

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/neuro-1977/BF6Portal-Tool.git
    cd BF6Portal-Tool
    ```

2.  **Install dependencies:**
    ```bash
    # Install Electron dependencies
    npm install

    # Install Web UI dependencies
    cd web_ui
    npm install
    cd ..
    ```

3.  **Run the Application:**
    ```bash
    npm start
    ```

## üíª Development

The project consists of two main parts:
1.  **Electron Main Process:** Handles the app window and file system operations (electron-main.js).
2.  **Web UI:** The Blockly editor, built with TypeScript and Webpack (web_ui/).

### Rebuilding the Editor
If you make changes to web_ui/src, you must rebuild the renderer bundles:

```bash
cd web_ui
npm run build
```

### Building for Production
To create a standalone executable (.exe):

```bash
npm run dist
```
The output will be located in the dist/ directory.

## üêç Python Utility Scripts

The `tools/` directory contains a suite of Python scripts designed to automate various maintenance, generation, and processing tasks related to the Blockly definitions and the overall project.

To run these scripts, navigate to the project's root directory in your terminal and execute them using `python tools/<script_name.py>`. Some scripts may require specific input files or have an implied order of execution.

### `update_blocks_db.py`
*   **Purpose:** Parses Blockly block definitions from TypeScript files (`web_ui/src/blocks/bf6portal.ts`, `web_ui/src/blocks/bf6portal_expanded.ts`) and the `web_ui/src/toolbox.ts` to create a `bf6portal_blocks.json` file. This JSON file acts as a central repository for all known block definitions and their properties.
*   **Usage:** `python tools/update_blocks_db.py`
*   **Output:** `bf6portal_blocks.json` (created in the project root).
*   **Prerequisite:** Should be run before scripts that consume `bf6portal_blocks.json`.

### `parse_blockly_js.py`
*   **Purpose:** Parses traditional JavaScript-based Blockly block definitions (if any) and converts them into a JSON-compatible format (`parsed_block_definitions.json`). This is useful for importing definitions from external JavaScript sources.
*   **Usage:** `python tools/parse_blockly_js.py`
*   **Input:** Assumes `web_ui/block_definitions.js` exists.
*   **Output:** `tools/parsed_block_definitions.json`.

### `generate_expanded_blocks.py`
*   **Purpose:** Reads `tools/parsed_block_definitions.json` (generated by `parse_blockly_js.py`) and dynamically generates/updates `web_ui/src/blocks/bf6portal_expanded.ts` with new block definitions.
*   **Usage:** `python tools/generate_expanded_blocks.py`
*   **Input:** `tools/parsed_block_definitions.json`.
*   **Output:** `web_ui/src/blocks/bf6portal_expanded.ts`.

### `fix_block_definitions.py`
*   **Purpose:** Ensures the consistency and correctness of Blockly block definitions in `web_ui/src/blocks/bf6portal_expanded.ts`. It categorizes blocks as Statement or Value and adjusts their `output`, `previousStatement`, and `nextStatement` properties accordingly.
*   **Usage:** `python tools/fix_block_definitions.py`
*   **Input/Output:** Modifies `web_ui/src/blocks/bf6portal_expanded.ts`.

### `generate_generator_stubs.py`
*   **Purpose:** Reads the block definitions from `web_ui/src/blocks/bf6portal_expanded.ts` and generates corresponding TypeScript generator stubs in `web_ui/src/generators/bf6_generators.ts`. These stubs are essential for converting Blockly blocks into executable code.
*   **Usage:** `python tools/generate_generator_stubs.py`
*   **Input:** `web_ui/src/blocks/bf6portal_expanded.ts`.
*   **Output:** `web_ui/src/generators/bf6_generators.ts`.

### `generate_toolbox_v2.py`
*   **Purpose:** Generates the `web_ui/src/toolbox.ts` file, which defines the categories and blocks that appear in the Blockly editor's toolbox. It reads block data from `bf6portal_blocks.json`.
*   **Usage:** `python tools/generate_toolbox_v2.py`
*   **Input:** `bf6portal_blocks.json`.
*   **Output:** `web_ui/src/toolbox.ts`.
*   **Prerequisite:** `update_blocks_db.py` should be run first.

### `fill_toolbox_gaps.py`
*   **Purpose:** Reads the current `web_ui/src/toolbox.ts` and `bf6portal_blocks.json` to identify any blocks present in the JSON data but missing from the toolbox. It then attempts to automatically insert these missing blocks into appropriate categories within `toolbox.ts`.
*   **Usage:** `python tools/fill_toolbox_gaps.py`
*   **Input:** `web_ui/src/toolbox.ts`, `bf6portal_blocks.json`.
*   **Output:** Modifies `web_ui/src/toolbox.ts`.
*   **Prerequisite:** `update_blocks_db.py` should be run first.

### `restore_toolbox.py`
*   **Purpose:** Reconstructs the `web_ui/src/toolbox.ts` file from JSON asset files (`web_ui/assets/<category_name>/<category_name>_data.json`) and `bf6portal_blocks.json`. This script is useful for rebuilding the entire toolbox structure from a defined set of assets.
*   **Usage:** `python tools/restore_toolbox.py`
*   **Input:** JSON asset files in `web_ui/assets`, `bf6portal_blocks.json`.
*   **Output:** `web_ui/src/toolbox.ts`.
*   **Prerequisite:** `update_blocks_db.py` should be run first, and JSON asset files should be present in `web_ui/assets`.

### `clean_bf6portal.py`
*   **Purpose:** A utility script to truncate the `web_ui/src/blocks/bf6portal.ts` file at a specific marker (`// Generator stubs`). This is used during development to manage the content of generated block definition files.
*   **Usage:** `python tools/clean_bf6portal.py`
*   **Input/Output:** Modifies `web_ui/src/blocks/bf6portal.ts`.

### `create_release.py`
*   **Purpose:** Creates a `.zip` archive of selected project files, intended for release distribution. (Note: The included file paths and release name may need updates for the current project structure).
*   **Usage:** `python tools/create_release.py`
*   **Output:** A `.zip` file in the project root.

### `catalog_helper.py`
*   **Purpose:** A helper class and script for cataloging Blockly blocks, potentially for documentation or other structured output.
*   **Usage:** `python tools/catalog_helper.py` (may require modification to its `main` function for specific use cases).

### `debug_json_extraction.py`
*   **Purpose:** A development utility for debugging the extraction and parsing of JSON-like structures from TypeScript files, particularly those defining Blockly blocks.
*   **Usage:** `python tools/debug_json_extraction.py` (primarily for internal developer use).
